## 一、分镜逻辑：「赶去见人」案例

**核心问题：** 单镜好看但连起来「跳」—— 分镜不是排列画面，而是建立因果。

### 错误 vs 正确分镜

**剧情：** 男主收到女主消息，赶去咖啡馆见她。

| 错误分镜（只画事件） | 正确分镜（补逻辑） |
|---------------------|-------------------|
| 收到消息 → 出门 → 走街上 → 坐到女主对面 | 收到消息 → 看时间意识到晚了 → 确认路线 → 出门赶路 → 路口红灯受阻 → 咖啡馆外确认 → 看见女主已等很久 → 坐下面对冷场 |

**错误版问题：** 画出了事件，没画出事件间关系。观众不知为何急、是否迟到、「老地方」在哪、女主为何冷脸。

### 四个逻辑补全点

#### 1. 补上「为什么急」

| ❌ 不要 | ✅ 要 |
|--------|------|
| 收到消息直接出门 | 收到消息后先看时间 → 观众知非普通赴约 |

#### 2. 补上「时间压力」

| ❌ 不要 | ✅ 要 |
|--------|------|
| 展示街景 | 快走+看手机 → 路口红灯车流拦住 → 制造「越急越不够」 |

#### 3. 补上「空间连接」

| ❌ 不要 | ✅ 要 |
|--------|------|
| 街道直接切咖啡馆内 | 先给咖啡馆外部 → 推门进入 → 外部与内部空间接上 |

#### 4. 补上「结果为什么成立」

| ❌ 不要 | ✅ 要 |
|--------|------|
| 女主冷淡无铺垫 | 咖啡不再冒热气 + 冷淡注视 → 观众知已等很久 → 冷脸情绪成立 |

### 分镜设计四问

设计每镜时除「下一镜画什么」，还须问：

1. **角色为什么行动？** — 站起、出门、奔跑都需理由
2. **角色要去哪里？** — 场景切换需明确目标
3. **中间有没有压力或阻碍？** — 「赶去见人」需时间压力
4. **结果为什么成立？** — 冷脸、尴尬等结果是前面逻辑的果

### Seedance 2.0 提示词结构

```
【基础设定】人物简单描述 + 场景描述
【氛围与画质】风格核心 / 视觉基调 / 色彩与影调 / 视觉基调参考
【画面内容】分镜N：景别 / 构图 / 运镜 / 画面内容 / 台词
```

| ❌ 不要 | ✅ 要 |
|--------|------|
| 画面依赖字幕、手机清晰文字 | AI 文字渲染不稳定，信息用画面行为传达 |
| 期望一次生成完整短片 | 多次生成 → 挑可用片段 → 剪辑拼合 |

**锚点：** 人物设定图 + 场景设定图作全能参考；逻辑由人设计，画面由 AI 生成。

## 二、复杂动作：两种方法

**核心：** 难在让 AI 理解动作前后关系，非生成漂亮单帧。

### 方法一：12 宫格原创分镜法

**适合：** 想自己设计镜头和动作，原创度高。

#### 第一步：人物 + 场景

- 确定发型、服装、体型、气质
- 风格化参考（如《蜘蛛侠：平行宇宙》漫画电影感）
- 电影截图 + 服饰参考图生图
- 多张电影截图垫图生成场景

#### 第二步：12 宫格分镜图

将人物图 + 需求交给 AI 扩写分镜 prompt：

- 12 宫格布局
- 指定风格参考
- 动作轨迹和步骤说明（如打篮球完整动作链）

#### 第三步：分镜 → 视频 prompt

AI 根据 12 宫格重新整理为 Seedance 2.0 可理解的动作描述 → 人物 + 场景 + prompt 生成。

| ❌ 不要 | ✅ 要 |
|--------|------|
| 一段长 prompt 描述复杂动作 | 先拆成分镜，再转视频 prompt |
| 期望一次出完整动作 | 多次生成，挑选正确片段拼接 |

**核心：** 先把复杂动作拆成分镜，再把分镜转成视频提示词。

### 方法二：视频参考替换法

**适合：** 已有动作优秀的参考视频，需快速稳定输出。

**操作：** 参考视频 + 人物图 + 场景图 → Seedance 2.0，强调画面风格不跑偏。

| 优势 | 注意 |
|------|------|
| 动作更稳定（完整运动节奏和镜头调度） | 网络参考视频仅学习测试；公开发布/商用需注意版权 |

**适合动作类型：** 篮球、跑酷、舞蹈、格斗等复杂肢体运动。

### 两种方法对比

| | 方法一 12 宫格 | 方法二 视频参考 |
|--|---------------|----------------|
| 原创度 | 高 | 低（替换角色场景） |
| 动作稳定性 | 需多轮筛选 | 高 |
| 推荐场景 | 有设计感的内容 | 快速得到稳定动作 |
| 组合策略 | 方法一打底 + 方法二提升动作稳定性 |

## 分镜一致性锚点

| 锚点 | 作用 |
|------|------|
| 角色/场景设定图 | 全能参考锁定外观 |
| 逻辑链（四问） | 镜头因果不断裂 |
| 12 宫格分镜 | 复杂动作顺序锚 |
| 参考视频 | 运动节奏锚（方法二） |
| 精简动作 prompt | 首尾帧模式不重复视觉描述 |

## 四、AI 长视频拼接（15s → 60s+）

> 溯源：[info-2862](https://www.super-i.cn/info-2862.html)（2026-06 站更）
> Prompt 层四法详解 → `prompt-director/references/video-prompt.md` §58
> 故事板拆分 → `prompt-director/references/director-storyboard.md` §57

**制片顺序：** 先设计衔接，再生成；不要生成完才想怎么拼。

| 方法 | 做法 | 成本 | 适用 |
|------|------|------|------|
| 视频续视频 | 上段尾 2–5s clip 作下段参考 | 高 | 动作/运镜须强连续 |
| 15s 故事板 | 60s 剧本 → 4 张故事板 → 逐段生成 | 中 | 多数叙事片（推荐） |
| 动作重叠 | 下段开头重复上段结尾情节 → 剪辑删重 | 低 | 已有多段需补救衔接 |
| 后期藏断点 | 删卡顿帧 + 叠化/模糊 | 最低 | 个人作品、教程 |

| ❌ 不要 | ✅ 要 |
|--------|------|
| 默认首尾帧硬接 | 评估运动惯性；尾帧在「运动中」时优先 clip 续写 |
| 60s 一次生成 | 按工具单次上限（通常 15s）拆段，每段有剧情边界 |
| 商用片全靠叠化 | 高要求项目在分镜阶段锁定衔接逻辑 |

**与分镜四问叠加：** 每 15s 段落内仍须补「为何行动 / 去哪 / 压力 / 结果为何成立」；段与段之间用故事板或重叠动作接。
