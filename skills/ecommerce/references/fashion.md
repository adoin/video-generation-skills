## FlowPix 通用模式：人·货·场自动化

多数服装工作流共享同一骨架，区别在 LLM 策划节点与输出形态。

| 模块 | 作用 |
|------|------|
| 模特定义 | 7 维标签：年龄/性别/人种/特点/头发/五官/气质 |
| 产品上传 | 白底平铺/挂拍，无需抠图 |
| LLM 造型师 | 识别品类 → 自动搭配 → 策划场景 |
| 批量出图/视频 | 分组执行，逐屏审核 |

### ❌ 不要

- 一上来点「整组执行」——难控画面，浪费积分
- 模特描述只写「好看女生」——人设漂移
- 视频生成前未确认静帧构图

### ✅ 要

- **逐组执行**：模特 → 换装 → 场景 → 视频，每步验收
- 平铺图背景干净、光线均匀
- LLM 锁定 Master Location，保证同批场景一致

## 详情页全流程

### 竞品拆解 → 结构策划（Lovart 对话流）

**适用：** 冲锋衣详情页、无实拍全案

| 步骤 | 动作 |
|------|------|
| 竞品透视 | 上传 2 张竞品长图，让 AI 拆视觉风格/布局逻辑 |
| 结构定稿 | 10 屏：超级符号 → 硬核科技(5屏) → 多维展示 → 品牌转化 |
| 首屏主视觉 | 明星+户外场景参考生成，Lovart「编辑文字」删多余文案 |
| 多维素材 | 三合一四状态图 + 自然动作（持水壶/调帽/插袋） |
| 黑科技可视化 | 写实 → 微距 → 白模 CG，水珠/透气物理效果 |
| 文案赋能 | 参数转社交语言（「暴雨？那是洗澡水」） |
| 排版交付 | 图片在 AI 完成，排版用 Figma/任意工具 |

### 白底图一键详情页（FlowPix）

| 步骤 | 动作 |
|------|------|
| 导入素材 | 上传白底图 + 自定义模特形象 |
| 文案控制 | 按屏填写卖点文案 |
| 逐屏生成 | 单节点执行，控制每屏画面 |
| 下载拼版 | 选满意图下载，外部排版 |

### ❌ 不要

- 说明书式参数堆砌
- 模特站桩无动作
- 首屏信息过载

### ✅ 要

- 长连页沉浸式场景（雪山/草甸，冷暖对比）
- 卖点图标化 + 微观科技展示
- Agent 对话迭代构图（「镜头再俯一点」）

## 主图 / Lookbook / Plog 批量

### SHEIN 级商拍主图

LLM 扮演「视觉总监」：识别单品类型 → 智能搭配 → 锁定主场景 → 输出：

- **4 张中焦**：截掉上半脸，视线聚焦衣服
- **2 张全身**：走路/插兜动态抓拍
- **6 张微距**：领口/袖口/面料纹理

### Lookbook 三阶段

1. **定制数字代言人** — 固定 Face ID
2. **单品变套装** — LLM 造型师输出搭配列表
3. **环境策划** — 5 套 look 方案选一，再生成

### Plog 九宫格（3-3-3 法则）

| 类型 | 数量 | 内容 |
|------|------|------|
| 产品 | 3 | 细节裁切、面料纹理 |
| 情绪 | 3 | 走路、回眸抓拍 |
| 氛围 | 3 | 咖啡/杂志/光影空镜 |

叠加 iPhone 胶片滤镜 + LLM 种草文案（标题/正文/标签）。

### Ins 平铺 / 黑底闪光灯 / 买家秀

| 工作流 | 输入 | 输出 |
|--------|------|------|
| Ins 平铺 | 单品图 | 路径 A 氛围流 / 路径 B 全套 OOTD Grid + 杂志排版 |
| 黑底闪光灯 | 模特描述 + 姿势参考 + 衣服图 | VOGUE 风排版或狗仔队闪光大片 |
| 海外买家秀 | 穿着图 → 提取白底 → 产品信息 + 风格词 | 咖啡厅/街景/居家多场景「真实买家」图 |

### 版型+纹样设计（TapNow）

分支一并行：5 版型 × 5 纹样 → 自选组合 → 复制节点融合生成。

### 节日营销物料

上传产品图 + 产品名 + 风格偏好（3D 渲染/手绘）→ 灵感提取 → 提示词整理 → 批量海报。

## 白底图 → 种草视频裂变

### 核心逻辑（LLM 指挥的视觉工厂）

```
平铺图 → LLM 策划 10 分镜(3全景+7微距) → 4K 静帧 → 物理动态视频 → 剪映合成
```

| 分镜类型 | 动态匹配 |
|----------|----------|
| 全景 | 缓慢推拉 / 失重悬浮 |
| 特写 | 面料起伏 / 光影流转 |

### 四风格生活方式裂变

一张白底图 → 4 套 Look（居家慵懒/街头工装/职场通勤/夜间探店）→ 对镜自拍视角 → 4 段视频 + 5 秒英文 Hook → CapCut 拼合。

### 素人种草全自动（6 段）

文本定义场景（「清晨日式客厅，短发活泼女孩」）→ 平铺图换装 → LLM 策划 6 个 Vlog 分镜 → 并行生成 → 剪映 + 文本朗读口播。

### 单张白底种草（5 片段）

上传白底图 + 产品卖点 → 工作流出 5 视频片段 → 剪映 + BGM。

### ❌ 不要

- 未确认分镜图就全开视频生成（成本高）
- 忽略物理材质描述（羊毛/棉/丝绸）

### ✅ 要

- 先跑通图片分镜，满意后再开视频开关
- 剪映适当变速 + BGM 卡点

## UGC / 口播 / 对镜拍

### 假人感破解原则

| 技法 | 说明 |
|------|------|
| 挡脸 | 对镜自拍，手机遮脸 |
| 不完美 | messy background / phone camera / natural lighting |
| 胶片颗粒 | 剪映叠加颗粒或自然光效，抹除 AI 锐利感 |

### UGC 全链路复刻

| 步骤 | 工具 | 要点 |
|------|------|------|
| 视觉解构 | Nano Banana Pro | 首帧反推提示词，多次迭代 |
| 精准换装 | 同上 | 自然语言描述视角，无损嵌入服装 |
| 动态赋能 | 可灵 O1 | 开幕动作迁移 + 图生视频 |
| 后期 | 剪映 | 胶片滤镜 + 营销文案轨道 |

### 对镜自拍工作流

1. **即梦 4.6** 生成底图（环境细节 + 穿搭质感）
2. **LLM 扩写** 批量裂变场景与动作
3. **以图推词** — 把满意图喂 AI 写视频动态提示词
4. **Veo 3.1 / 即梦 3.5pro** 成片

### 低头杀 + 对镜拍（FlowPix）

造型组（模特+产品融合）→ 摄影组（俯身 POV + 对镜蹲拍，同人同衣）→ 首尾帧生成视频 → 音效组 → 剪映合成。

### 连贯口播（首尾帧接龙）

| 步骤 | 动作 |
|------|------|
| 人物灵魂 | Gemini 3 Pro 将需求转标准化人设参数 |
| 产品植入 | 换装确保 Logo/材质/抽绳与实物一致 |
| 关键帧脚本 | 图1-图7 连续动作（看镜头→持咖啡→指 Logo→推近领口→袖口→拉帽→戴帽） |
| 首尾帧桥接 | [图N]→[图N+1] 逐段生成，锁死一致性 |
| 剪辑 | 片段首尾相连，一镜到底错觉 |

### 会说话博主（可灵 2.6）

Gemini 写镜头语言 → Midjourney 多场景底图 → 可灵 2.6 图生视频，口播用 `""` 标出 → 剪映字幕。

### 美妆 UGC 首尾帧接龙

脚本：震惊开头 → 拿产品 → 展示质地 → 涂抹 → 满意结尾。Nano Banana 关键帧序列 + Veo 3.1 逐段接龙。

## Seedance 2.0 服装视频

### 通用操作

- 模式：**全能参考**
- 素材：服装图 + 可选模特图
- 提示词：**英文 + 时间轴分幕**

### 五种风格模板

| 风格 | 定位 | 素材 |
|------|------|------|
| Ins 奶油色教练夹克 | 复古叠穿种草 | 单品+裤子+鞋+耳机 |
| 北美 TK 棒球衫 | 垂坠感+复古细节 | 服装图+模特图，提示词指定换裤 |
| 抖音电商 CONVERSE | 鞋子质感+搭配 | 鞋图为主，检查无字幕干扰 |
| 视频二次修改 | 元素替换 | 参考视频+环境图+替换物 |
| 分镜式 Ins | 九宫格叙事 | Nano Banana 生分镜图 + 原图 |

### 变装两条路径

| 路径 | 流程 |
|------|------|
| Lookbook | 模特底图 + 换装 → 即梦逐套生视频 → Look 1/2/3 转场 |
| 一镜到底 | 底图+穿搭图+参考视频 → Seedance 直出 → 剪掉不合格帧 |

### ❌ 不要

- 中文提示词（海外风格案例）
- 忽略音乐节奏适配

### ✅ 要

- 检查转场流畅度与画面无文案干扰
- 排队长时可用 Veo 3.1 替代

### 服装工厂宣传（三张图工作流）

**铁律：**
- 标注图1/2/3 职责，老板须融合进工厂环境

- 老板须有具体动作（边走边讲/检查走线/嘴部开合）

## 机票 × 城市穿搭转场（创意种草）

> 溯源：[info-2883](https://www.super-i.cn/info-2883.html)（2026-06 站更）
> 创意广告向亦见 `brand-ad-cg/references/creative-ads.md`

**核心逻辑：** 每张机票 = 一个目的地 = 一套穿搭；机票设计 → 人物 → 动态 → 剪映模板合成。

### 七步流程

| 步骤 | 动作 |
|------|------|
| 1 机票结构 | 截参考机票 → Image 2.0 去人物 → LLM 分析**层级**：纸质感 → 大字地名 → 人物层 → 小字信息层 |
| 2 城市主题 | LLM 推荐城市（伦敦/巴黎/夏威夷/上海等）+ 对应穿搭气质 |
| 3 穿搭参考 | 小红书搜风格关键词 → 下载 → 提取服装/发型/配饰（非直接用原图） |
| 4 人物素材 | 固定脸模三视图 → Image 2.0 假人白底展示 → Nano Banana Pro 脸+服装合成（加负面：假脸/塑料皮/头身比） |
| 5 视频动作 | LLM 据首图推运镜/小动作 → Seedance 2.0（拉远、转头、裙摆、持物等） |
| 6 多色机票 | **保留版式只改色** — Image 2.0 图生图；色板跟穿搭（英伦暖米/巴黎雾粉/夏威夷海盐蓝等） |
| 7 剪映 | 机票去小字 → 透明底 → 模板替换素材 → 跟音乐调入场/缩放/转场 |

| ❌ 不要 | ✅ 要 |
|--------|------|
| 让 AI 重新设计机票版式 | 强调「大字底层、小字上层、预留人物位」 |
| 每套不同脸 | 先锁脸模再三视图换装 |
| 人物站桩 | 每段加轻微动态，防抠图感 |

## 一周开车穿搭（固定视角裂变）

> 溯源：[info-2890](https://www.super-i.cn/info-2890.html)（2026-06 站更）
> 工具入门交叉 → [info-2796](https://www.super-i.cn/info-2796.html)

**适用：** TikTok/小红书「Monday–Sunday 穿搭」系列；一张开车视角模板裂变 7 套。

### 六步流程

| 步骤 | 工具 | 动作 |
|------|------|------|
| 1 基础视角 | Image 2.0 | 固定「车内开车 POV」结构（先不对穿搭） |
| 2 换脸 | Nano Banana Pro | 统一授权/虚拟模特脸（**勿未授权真人**） |
| 3 七套服装 | 白底穿搭图 ×7 | 与开车图一起上传 → 逐套换装 |
| 4 视频 prompt | Gemini 等 LLM | 上传首图 + 模板 → 推导每套动态 prompt |
| 5 生成 | 可灵 O3 | 图 + prompt 逐条生成 ~7 段 |
| 6 剪辑 | 剪映 | 按 Mon–Sun 排列 + 星期贴纸 + 滤镜/BGM |

| ❌ 不要 | ✅ 要 |
