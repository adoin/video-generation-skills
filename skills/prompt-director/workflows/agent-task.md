# Agent 任务拆解（§61 · GCSD）

> 溯源：[info-2886](https://www.super-i.cn/info-2886.html)（2026-06 站更）
> 与 confirmation-gates 互补：闸门收集**创作设定**；本文管**可执行任务**写法。
> 情绪短片 Codex 案例 → `brand-ad-cg/references/tvc-ads.md` · `ai-video-director/narrative/film-style.md` §五

## 何时用

用户要把模糊想法推进到**多文件交付**（剧本、分镜、素材清单、粗剪工程），或要串联多个 skill / 工具自动执行时——不要写一段「帮我做高级一点」的长 prompt，改写 **GCSD 任务单**。

## GCSD 四要素

| 字母 | 含义 | 示例 |
|------|------|------|
| **G** Goal | 最终交付什么 | 60s 竖屏古风短剧分镜 + 逐镜 seedance prompt |
| **C** Context | 读哪些资料 | 角色三视图文件夹、参考视频、品牌色板 md |
| **S** Steps / Skill | 调哪个 skill、按何规则 | `ai-video-director/short-drama` + `prompt-director` §59 |
| **D** Done | 验收标准 | 6–8 镜故事板 md、无重复镜头、每镜 ≤15s prompt |

| ❌ 不要 | ✅ 要 |
|--------|------|
| 「做得高级一点」「节奏好一点」 | 「60s 竖屏 / 前三秒出现冲突 / 不重复镜头 / 输出分镜 md」 |
| 一次要求最终成片 | 分阶段验收：人物卡 → 分镜 → 单镜 prompt → 视频 |

## 任务五层（Codex / Agent 通用）

1. **本次目标** — 这一次具体做什么
2. **原始资料** — 文件、图片、视频路径
3. **固定方法** — Skill / reference 里的规则（人物卡格式、三幕、镜头表）
4. **执行工具** — 文件读写、浏览器、剪辑软件、生图 API
5. **验收标准** — 交付物清单 + 必查项（黑帧、断音、脸漂移）

**分工：** 单次任务消息 =「这次做什么」；Skill =「这类任务永远怎么做」。

## 多 Skill 编排（短剧示例）

上传多个 `.md` skill 后，一句话触发流水线：

| Skill 职责 | 输出 |
|-----------|------|
| 编剧 | 题材、人物关系、分集目录、完整剧本 |
| 人物 | 身份、妆造、白底定妆、图生视频风格 |
| 场景 | 空间九宫格、陈设、光线 prompt |

**操作顺序：**

1. 上传 skill 文件 → 说故事想法 → 迭代剧本
2. 剧本 OK → 生成人物/场景 prompt（**强调白底定妆照**）
3. 分镜脚本 → seedance 逐镜视频 prompt + 氛围/色板 prompt
4. 分阶段验收，勿跳步要成片

## 粗剪与自动化边界

| 能力 | 适合 | 不适合 |
|------|------|--------|
| Agent 读文件夹 + 写剪辑清单 | 排序、命名、删黑屏/重复 | 情绪卡点、高级节奏 |
| 操控剪辑软件鼠标粗剪 | 第一版时间线、演示自动化 | 宣传为「一键专业成片」 |

**铁律：** 验收看**可检查项**（文件是否存在、镜号是否连续、参考是否标注），非主观形容词。
